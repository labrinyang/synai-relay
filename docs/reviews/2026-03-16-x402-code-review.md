## CRITICAL

### C1. No server-side amount/payTo verification in `create_job_endpoint`
- **File**: `server.py:1232`
- **Issue**: `server.verify_payment(payload, payload.accepted)` passes the **client's claimed** payment requirements to verify/settle. An attacker can craft a payment header with `amount="1"` (0.000001 USDC) and `payTo=<attacker_address>`, creating a `funded` job with full escrow status while paying nearly nothing.
- **Compare**: The submission paywall at `server.py:1769-1775` correctly validates `payload.accepted.amount == expected_atomic`.
- **Impact**: Funds loss — attacker creates arbitrarily-valued funded jobs for free.
- **Fix**: Build server-side requirements, assert `amount` and `pay_to` match before verify/settle.

### C2. `_auto_refund()` bypasses ChainAdapter — always uses Base L2 WalletService
- **File**: `server.py:456,471,1930`
- **Issue**: All `_auto_refund()` call sites pass `get_wallet_service()` (Base L2) directly, ignoring `job.chain_id`. For X Layer jobs (`chain_id=196`), refund attempts on wrong chain.
- **Compare**: Payout path at `server.py:697` correctly uses `_chain_registry.get_or_default(job.chain_id)`.
- **Impact**: Funds loss for non-Base chain jobs on refund.
- **Fix**: Refactor `_auto_refund` to accept a `ChainAdapter` or look up via `_chain_registry`.

### C3. `OKXFacilitatorClient.get_supported()` returns raw `dict`
- **File**: `services/okx_facilitator.py:68-69`
- **Issue**: Returns `self._client.get(...)` (a `dict`). The x402 SDK protocol expects `SupportedResponse` object with `.kinds`, `.signers`. Crashes on `x402ResourceServerSync.initialize()`.
- **Impact**: OKX facilitator registration fails at startup. Blocks X Layer.
- **Fix**: Parse response into `SupportedResponse`.

### C4. `OnchainOSClient.get()` — query params not URL-encoded
- **File**: `services/onchainos_client.py:56-58`
- **Issue**: `"?" + "&".join(f"{k}={v}" for ...)` — no encoding. Special chars (`&`, `=`, spaces) break URL and HMAC signature.
- **Impact**: Correctness bug + potential injection.
- **Fix**: Use `urllib.parse.urlencode(params)`.

## IMPORTANT

### I1. `x402_service.py` imports `PaymentRequirements` unconditionally
- **File**: `services/x402_service.py:5`
- **Issue**: `from x402 import PaymentRequirements` at module top level. `server.py:40` imports from `x402_service` outside the `try/except`. If x402 SDK is absent, the **entire server fails to start** — contradicts `_X402_SDK_AVAILABLE` graceful-degradation design.
- **Fix**: Wrap import in `try/except` or move import inside `build_requirements()`.

### I2. `PAYMENT-RESPONSE` header never set after settlement
- **File**: `server.py:1246-1257` (create_job), `server.py:1790-1803` (get_submission)
- **Issue**: x402 spec requires `PAYMENT-RESPONSE` header with base64-encoded settlement receipt. `encode_payment_response_header` is imported at `server.py:33` but never called.
- **Fix**: Add header to 201/200 responses after successful settlement.

### ~~I3. Submission view payments go 100% to worker — no platform fee~~
- **Status**: Resolved — intentional design decision
- **Decision**: 100% to worker. Platform earns from the 20% job escrow fee. Paying workers directly for solution views incentivizes quality submissions.

### I4. `settle_result.payer` can be `None`
- **File**: `server.py:1253`
- **Issue**: `depositor_address=settle_result.payer` stores `None`. Makes on-chain refund impossible (guard at line 2025 checks `job.depositor_address`). Job degrades to `off-chain` refund with no actual funds return.
- **Fix**: Guard against `None` payer — reject settlement or log warning.

### I5. `_ensure_x402_init` not thread-safe
- **File**: `server.py:266-271`
- **Issue**: Plain boolean flag without lock. Concurrent first requests could run `_init_x402()` multiple times.
- **Fix**: Use `threading.Lock` or `threading.once`.

### I6. `BaseAdapter.refund()` — no exception handling
- **File**: `services/base_adapter.py:49-51`
- **Issue**: `self._ws.refund()` called bare. Exception propagates unhandled. Other methods (verify_deposit, payout) handle errors gracefully.
- **Fix**: Wrap in try/except, return `RefundResult(error=str(e))`.

### I7. `OKXFacilitatorClient` — `error_reason` and `error_message` both map to `errorMsg`
- **File**: `services/okx_facilitator.py:59-65`
- **Issue**: Both fields populated from `data.get("errorMsg")`. x402 SDK defines them as distinct (machine-readable vs human-readable).
- **Fix**: Map `error_reason` to `data.get("errorCode")` or `data.get("failReason")` if OKX provides it.
